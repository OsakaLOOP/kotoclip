# 悬浮词典气泡

## 数据流

气泡查询返回查询词、读音、假名表记候选、已选目标和按词典来源排序的词条。`@@@LINK` 在后端解析到实际词条；正文中的 `entry://` 不交给系统浏览器，而是提取为带关系类型的受管理导航。

纯假名导航页会保留全部表记候选。用户选择写入画像库的 `user_dictionary_choices`，并在画像引擎启动时载入内存；再次查询相同“假名 + 读音”时优先展示已选词条。选择只影响词典展示，不改写原文或分词结果。

## 交互边界

正文胶囊与气泡分别维护悬浮状态。离开正文后使用短延迟允许指针跨越间隙进入气泡；进入气泡即取消关闭，离开两者后才关闭。每次异步查询带前端请求序号，旧查询不能覆盖当前词条。

气泡具有独立滚动区域、词条内返回历史、粘性词头和词典分组标题。滚轮事件不会冒泡到阅读器。纯假名导航、亲项目、子项目、参照、近义和反义链接从原 HTML 中移除，分别由候选选择器和关系分组组件渲染。词典 HTML 只保留词头、重音、词性、分义、释义和例句等正文，经后端白名单清洗后使用大辞林专用内部样式；不会加载词典携带的外部 CSS。

## 多词典扩展边界

每个词条具有稳定的 `entry_key`、`dict_name`、匹配类型和关联链接。后端继续扫描 `data/dicts` 下所有兼容 SQLite，并按用户优先级稳定排序；前端按 `dict_name` 分组，因此增加词典不需要改变气泡协议。

## 真实内容审计

以下命令对当前大辞林做分层抽样，并验证第一话中的代表形态、重定向和纯假名多候选：

```powershell
python scripts/audit_dictionary_bubble.py --dict data/dicts/daijirin.db --source "D:\Downloads\epub-exp\source\七日の喰い神 (ガガガ文庫) (カミツキレイニー)\output.md"
```

审计覆盖 `ボリューム`、`ひやし`、`七日` 的重定向，`いる`、`ある`、`かみ` 的一对多表记，以及词典 HTML 标签和内部链接的分层样本。单元测试另覆盖链接清洗、目标跳转及用户选择重启后的持久化。
