## 验收口径

三个阶段都必须提供命令行入口、真实文本证据、自动化测试和独立提交。机械完整性只验证文本及范围契约，不替代分词或语法的人工语义判定。

## 阶段一：外部词典与覆盖率

状态：完成。

### 根因

1. 开发运行只加载 AppData 中的 `starter.sqlite`，没有加载仓库的 `daijirin.db`。
2. 大辞林原始词头采用 `けいさつしょ【警察署】` 格式，旧查询却用 `警察署` 做等值查询。
3. 实际 `daijirin.db` 仍是旧 schema，缺少 `reading` 列和结构化检索索引。
4. 假名跳转词条会返回 `@@@LINK` 占位内容；异体字 `繋／繫` 也会迫使查询退化为不可靠的同音匹配。

### 实现

- schema v3 保留 `entries.headword` 原始 MDX 键，并新增：
  - `entry_forms`：汉字、假名、混合及其他表记；
  - `entry_readings`：原始读音和统一片假名读音。
- 查询顺序：规范化表记 → 纯假名读音 → 原始词头 → 显式读音 → 有限 FTS。
- 跳过 `@@@LINK` 占位项；支持 NFKC、平/片假名、分隔符和实证异体字 `繋／繫`。
- `contains_exact` 使用同一表记索引，使 `警察署`、`はぐれ者` 等词汇边界判断真正依赖外部词典。
- 开发构建加载仓库 `data/dicts`；安装构建仍使用应用数据目录。
- `kotoclip-cli` 提供 `dict-info`、`lookup`、`analyze`、`audit`、`repl`。

### 全章实测

| 非空分析行 | 1142 |
| 分析字符 | 40046 |

| 读音命中 | 810 |
| 未命中 | 404 |
| 词典覆盖率 | 95.73% |
| 文本重建通过率 | 100% |
| 范围机械完整性 | 100% |

未命中中，`ラティメリア` 137 次、`マガツカミ` 91 次，二者均为作品专名。其余主要是拟声、拉长口语、外来专名和跨词组合。

### 尚未证明的事项

- 95.73% 是词汇出现级词典覆盖率，不是语法语义准确率。
- 语法和分词的 95% 可靠性需要阶段二的跨文节表达模型及人工判定 CLI 样本共同证明。
