# Kotoclip 分析研究进度

更新日期：2026-07-11

## 验收口径

三个阶段都必须提供命令行入口、真实文本证据、自动化测试和独立提交。机械完整性只验证文本及范围契约，不替代分词或语法的人工语义判定。

## 阶段一：外部词典与覆盖率

状态：完成。

### 根因

1. 开发运行只加载 AppData 中的 `starter.sqlite`，没有加载仓库的 `daijirin.db`。
2. 大辞林原始词头采用 `けいさつしょ【警察署】` 格式，旧查询却用 `警察署` 做等值查询。
3. 实际 `daijirin.db` 仍是旧 schema，缺少 `reading` 列和结构化检索索引。
4. 假名跳转词条会返回 `@@@LINK` 占位内容；异体字 `繋／繫` 也会迫使查询退化为不可靠的同音匹配。

### 实现

- schema v3 保留 `entries.headword` 原始 MDX 键，并新增：
  - `entry_forms`：汉字、假名、混合及其他表记；
  - `entry_readings`：原始读音和统一片假名读音。
- 查询顺序：规范化表记 → 纯假名读音 → 原始词头 → 显式读音 → 有限 FTS。
- 跳过 `@@@LINK` 占位项；支持 NFKC、平/片假名、分隔符和实证异体字 `繋／繫`。
- `contains_exact` 使用同一表记索引，使 `警察署`、`はぐれ者` 等词汇边界判断真正依赖外部词典。
- 开发构建加载仓库 `data/dicts`；安装构建仍使用应用数据目录。
- `kotoclip-cli` 提供 `dict-info`、`lookup`、`analyze`、`audit`、`repl`。

## 阶段二：跨文节表达

状态：类型化统一重构完成，进入人工实例验证。

### 2026-07-12 重构

- 用户、内置、词典和非连续候选进入统一表达分析出口，并按类型、优先级、置信度和范围排重。
- 正式类型分为 `lexical_unit`、`idiom`、`grammar_construction`、`correlative`；只有词汇单位可以重组 token 和文节。
- 用户规则保存四级词性、活用类型、活用形、文节内部范围和组合方式；复用时不再只比较字符串和一级词性。
- 自动词典扫描增加组合方式硬约束，以助词或助动词起始的 `につく`、`とする` 一类普通句法拼接不再直接成为词典表达。
- 扁平流保留标点和换行边界；非连续呼应优先最近相容尾项，不再跨句、跨段或跨对话寻找最远尾项。
- 实例规则 modal 支持选择四类表达、编辑标签和说明、调整优先级、配置固定/词性槽位/任意文节、选择语素范围及插入非连续间隔。
- 词汇单位合并后仍保留全部底层 morpheme；表达标签可以按整体表层形发起词典查询，双击具体语素会按其自身辞书形和读音下钻。旧的“拆为形态素”入口已移除，右键菜单保留 N-best 分词候选。

下一版本再研究从词典正文提取品词、活用和惯用句标记，以及多词典、多候选、空导航词条的消歧；本版本未接入词条详情证据。

### 后期任务路线

完整任务、依赖和验收标准见 `docs/cross_bunsetsu_expressions.md` 第 13 节。执行顺序为：

1. 完成 341 项类型化审计和固定正反例回归基线。
2. 建立词典结构证据、多候选消歧和统一组合方式 DSL。
3. 将词汇单位与 lattice/N-best、局部文节和画像联合决策。
4. 深化非连续边界域、命名捕获和 span graph 冲突解析。
5. 完整实现实例规则预演、词典绑定、字段级约束和上下文负反馈。
6. 完成规则目录版本治理、表达画像、导出和批量性能诊断。

阶段二不能仅以命中数验收。最终口径同时包括分类准确率、边界决策准确率、整体释义可追溯率、硬边界越界数、文本重建完整性和用户撤销可逆性。

- 历史 `user_merge_rules` 已退出正式分析；Vibrato 文节和词典边界不再被旧拖拽记录重写。
- 内置表达目录在跨文节语素流上运行确定性状态机，返回精确字符范围、token 范围、类别和整体释义。
- 用户可拖拽两个以上文节，补充短标签、整体含义和显式可变槽位；规则持久化到独立 SQLite 表。
- 可变槽位匹配辞书形以外的内容，但仍约束词性和助词序列；时态、礼貌形及句末语气经过结构化归一。
- UI 使用细连接带、局部点状下划线和短标签，不合并或放大原文节胶囊；规则侧栏支持查看和删除。
- CLI 新增 `expression-preview`、`expression-scan`、`expression-add`、`expression-list` 和 `expression-repl`。

### 第一话交互扫描

指定章节共发现 10 个跨文节命中、9 种表达：

- `有無を｜言わさず`
- `休む｜間もなく`
- `食料に｜すぎない`
- `にも｜かかわらず`
- `こと｜なく`（2 处不同上下文）
- `に｜しては`
- `何が｜何でも`
- `目も｜くれず`
- `なすが｜ままに`

`ない限り`、`からには` 等当前已位于单一文节的语法不进入本层。`どんなに…たところで` 等非连续结构留给阶段三 NFA/状态机候选。

## NLP 管线进度 UI

状态：实现完成，待 UI 手动确认。

- Core 在准备、形态素、文节、语法、词典边界、画像评分、曝光记录和完成阶段报告结构化进度。
- 词典、画像和曝光阶段按实际 token 数报告 `completed/total`；百分比只表示真实阶段权重，不按计时器虚构。

- `benchmark` 命令使用完整 Engine 和真实曝光写入记录各阶段耗时。

### 交互观察

- `七日`：V1 为 `七｜日`（cost 781），整体词 `七日／ナノカ` 为 V2（cost 5706）。外部词典证据可将 V2 提升为推荐，但不改写原始成本。
- `警察署`：lattice 中没有整体节点，N-best 不能生成词典不存在的词；继续由外部词典边界层处理。
- `取り調べ`：动词路径与名词路径成本只差 358，是有语境判定价值的 POS 歧义。
- `有無を言わさず`：候选包含 `言わす＋ぬ` 与 `言う＋す＋ぬ` 等真实路径，可用于人工比较。

### 后续策略

原始 lattice N-best 已可用。下一轮再组合两类证据：Vibrato 成本与词典/表达状态机重排；推荐结果保留人工选择和撤销入口，不直接覆盖原始 rank。非连续表达如 `どんなに…たところで` 由独立 NFA 处理，不伪装成分词候选。
