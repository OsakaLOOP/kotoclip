状态：已实施并通过代表样本验收

日期：2026-07-22

证据目录：

- 重构后：`.agents/analysis/dictionary-form-matrix-after-20260722/README.md`

本文记录表记矩阵重构的最终决策、迁移结果和验收结论。当前运行时协议由
`dictionary_lookup_and_bubble_refactor.md` 与 `dictionary_internal_architecture.md`

## 1. 范围

本次重构集中处理确定性的查询路由、表记切换、条件门和状态协议：

- 将表记、词典和 occurrence 拆成三个正交选择轴；
- 以全局固定词典列表示可用性；
- 合并跨词典的相同规范表记；
- 保留归一组内的原始形式、证据和优先级；
- 逐词典执行查询降级，避免一本词典的命中改变其他词典的路由；
- 将 navigation/redirect 限定为表记发现与可用性校正证据；
- 使表记切换只更新活动正文。

语境排序、词频裁剪、词汇等价判断、跨词典 sense 对齐和综合释义留给后续阶段。

## 2. 最终对象模型

日语词典查询中的对象分为四层：

1. 正文或形态链提供的观察形式、读音和词性；
2. 可选择的全局表记组；
3. 某个“表记 + 词典”单元格内的 occurrence；
4. occurrence 内部的 sense 树。

气泡使用以下选择关系：

```text
活动表记
  × 活动词典
    × 当前单元格 occurrence
      → occurrence 内部 sense 树
```

相同规范表记跨词典无条件合并为一行。读音条件先筛选 occurrence，再由各词典单元格
保存词性、源记录拆分和义项数量。该规则使 `行く` 只占一个表记行；`イク` 查询只装配
`いく`，`ユク` 查询只装配 `ゆく`。

全部已启用词典构成固定列，顺序服从用户词典优先级。某个单元格缺少实质正文时，
该词典在当前表记下暗显；表记行和其他词典列保持稳定。

## 3. 查询协议

请求结构：

```text
DictionaryLookupRequest
├─ query
├─ observed_form?
├─ reading?
├─ pos?
├─ selected_form_id?
└─ dictionary_order[]
```

响应结构：

```text
DictionaryLookup
├─ query
├─ observed_form?
├─ reading?
├─ pos?
├─ mode
├─ selected_form_id?
├─ forms[]
│  ├─ form_id
│  ├─ display_form
│  ├─ normalized_form
│  ├─ readings[]
│  ├─ evidence[]
│  ├─ score
│  ├─ variants[]
│  │  ├─ surface_form
│  │  ├─ readings[]
│  │  ├─ evidence[]
│  │  ├─ score
│  │  └─ dictionary_names[]
│  └─ dictionaries[]
│     ├─ dictionary_name
│     └─ available
├─ dictionary_names[]
├─ entries[]
└─ timing?
```

`query/observed_form/reading/pos` 回显根条件。`forms[]` 与 `dictionary_names[]` 表达全局
矩阵。`entries[]` 只承载活动表记的已加载 occurrence；前端再按词典划分活动单元格。

## 4. 逐词典路由

每本词典独立执行以下阶段：

1. 查询精确表记；
2. 读取该查询的 alias 与 navigation/redirect 目标；
3. 精确结果缺少实质正文且 alias 缺少兼容记录时，按兼容读音回退；
4. 应用读音和记录类型准入门，形成表记种子；
5. 用精确索引探测候选表记在该词典中的可用性；
6. 只为活动表记加载并适配正文。

准入门接受精确表记及具有兼容读音证据的 alias/读音结果。显式读音和纯假名 query
推导出的读音同时约束表记读音、词典单元格与活动正文。声明了其他读音的 occurrence
会被排除，缺少读音的记录继续依赖表记与实质正文证据。例如 `なれる` 不接收
`縄/名和/那波`，`強い/コワイ` 不接收 `つよい`。

navigation/redirect 记录提供普通表记目标和索引证据，不进入 `entries[]`。因此 UI
无需维护独立的跳转状态，切换表记始终通过同一个根请求和 `selected_form_id` 完成。

## 5. 归一化与原始形式

归一化只生成分组 key。每个输入形式保存在 `DictionaryFormVariant` 中，并携带：

- 原始 `surface_form`；
- 适用读音；
- 发现证据；
- 排序分数；
- 来源词典。

`display_form` 根据观察形式与证据优先级选择。加载活动表记时，查询器会加载组内全部
variants，再按 occurrence ID 去重。字符或脚本归一不会覆盖词典原始形式，也不会抹去
各形式的排序信息。

variant 的 `dictionary_names[]` 记录该原始形式在当前读音条件下的发现来源。规范等价后的
单元格可用性由精确加载、读音条件和实质正文共同确认，不会反向改写 variant 来源。

当前确定性规则覆盖 NFKC 兼容字符、平假名/片假名比较、词典明确列出的表记、全汉字
备选串和适配器识别的可选送假名。相同读音本身不建立汉字表记等价关系。

## 6. 状态与交互

| 状态 | 所有者 | 生命周期 |
| --- | --- | --- |
| 根查询、观察形式、读音、词性 | 解释会话 | 当前解释根 |
| 表记矩阵 | Lookup 响应 | 根查询不变期间 |
| 活动表记 | 解释会话 | 当前气泡 |
| 活动词典 | Tooltip | 当前气泡 |
| occurrence 选择 | Tooltip | 当前表记与词典单元格 |
| 词典优先级 | Profile | 跨会话 |

表记选择不写入画像，也不进入普通正文关系历史。活动表记与活动词典共同解析为可用
单元格：选择一轴时保留兼容的另一轴，当前组合不可用时按全局顺序回退。暗显项保留并
可触发联动；整个查询都没有可用表记的词典才禁用。occurrence 选择以
`form ID + dictionary name` 为作用域。

表记不超过 8 项时使用紧凑选择条，超过 8 项时进入集中菜单。菜单保留全部表记和当前
选中项，避免横向列表挤压正文。

## 8. 验收结果

| 场景 | 结果 |
| --- | --- |
| `行く / イク` | 一个 `行く` 表记行；四本词典共 4 个 `いく/イク` occurrence |
| `いく / イク` | 11 个有确定证据的表记行；`いく/イク` 两个原始 variant 均保留 |
| `私 / ワタシ` | 三本词典各一个 `わたし`；`わたくし/し` 等冲突读音不进入正文 |
| `強い / コワイ` | 一个 `強い` 表记行；大辞林、小学馆各一个 `こわい`，Crown 不可用 |
| `する / スル` | 10 个有确定证据的表记行；`刷る` 在三本正式词典中可用 |
| `なれる / ナレル` | 保留兼容读音表记；`縄/名和/那波` 被拒绝 |
| `がんばる / ガンバル` | 同分时按词典覆盖数选择 `頑張る`；`眼張る` 继续保留 |
| 表记切换 | `いく→行く`、`する→刷る`、`わたし→私`、`あの→彼の` 的矩阵稳定 |
| 双向路由 | `あの + 大辞林` 自动解析为 `彼の + 大辞林`；`彼の→あの` 回退到优先可用词典 |
| 全部固定样本 | 每行 4 个词典单元格；活动正文没有冲突读音或 navigation/redirect |

重构后目录包含 24 组 JSON/HTML 样本，其中四对验证表记切换前后的矩阵稳定性。
机器检查确认每个活动行的可用词典集合与正文词典集合一致。

## 9. 外部模型校准

JMdict 将表记、读音和 sense 分层，并使用 `re_restr`、`stagk`、`stagr` 表示它们之间
的适用范围。Unicode NFKC 处理规范与兼容等价，无法判断日语词汇等价。文化厅
《送り仮名の付け方》列出的本则、例外和许可形式也说明送假名变化需要独立证据。
相关来源登记在仓库根目录 `sources.md`。

## 10. 后续边界

纯假名高同音查询仍可能返回较多表记行。当前实现保留具有明确索引、alias 或兼容读音
证据的结果，维持可解释、可复现的路由。后续可在现有矩阵上增加语境评分、词频信号、
领域类型过滤、词汇等价图和跨词典 sense 对齐；这些能力应保持原始 variants、固定词典列
与单元格 occurrence 身份稳定。
