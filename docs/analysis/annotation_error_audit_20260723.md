## 1. 范围与验收口径

本轮输入来自 `D:\DCMT\dev notes\未命名 9.md`、其中 10 张截图、Kotoclip
书库中的原始 EPUB 转录文本，以及当前 `kotoclip-cli` 的实际输出。审计使用 2026-07-23
的 `master`（`d84ab44`）重新编译 CLI，不把截图中的旧界面表现直接当作当前后端结果。

统一复现入口：

```powershell
cargo build -p kotoclip-core --bin kotoclip-cli
target\debug\kotoclip-cli.exe analyze --text "正文"
target\debug\kotoclip-cli.exe grammar-inspect --text "正文"
target\debug\kotoclip-cli.exe lookup --word "词面" --reading "读音" --full
```

本轮按以下所有权验收正文颜色和查询目标：

- 黄色／画像色词汇核心吸收同一词汇的活用、否定、时态、语态，以及 `〜ている／〜ていた`
  的体貌变化；这些范围仍可通过语法 badge 查看说明，但不应在正文中把一个谓词切成黄蓝两半。
- `〜てくれる／〜てみせる` 的补助用言、`ごとき／べき／ものの` 等相对独立的功能成分保留为
  蓝色语法范围；其自身活用必须完整落在同一个蓝色范围内。
- 整体词、生产型复合词和惯用语必须分别由词典词汇层、构词层和表达／语法层负责；文节不能仅因
  字符串词典命中而吞并普通句法。
- 未覆盖项保持可审计，不以“把所有助词染蓝”代替构式识别。

状态：`待修` 表示已复现；`对照` 表示截图或摘录所示行为在当前版本正确；`待语境` 表示词面本身
多义，必须用后续真实句子确定；`已修` 只在对应提交和回归均完成后使用。
