# 词典释义细节与例句交互审计

日期：2026-07-19

基线：`654108a refactor(dictionary): unify multi-dictionary lookup and bubble`

范围：小学馆日中辞典、三省堂 Super 大辞林 3.1、Crown 日中辞典的原始 HTML、适配器 IR、CLI 结构化预览与 Vue 气泡组件。

## 1. 审计目标

本次审计只处理直接影响可读性与可操作性的细节：

1. 小学馆释义中的括号、逗号、分号、内部日文限定和行内标签；
2. 同一义项内多个 `〔日文范围〕` 解释组的归属；
3. 三省堂词性标签、`<small>` 泄漏、可能/派生/惯用/表记模块串入正文；
4. 同形同音 occurrence 切换标签错误读取义项正文，造成正文重复和胶囊溢出；
5. Crown 的日文范围与中文释义不必要换行；
6. 三本词典例句超过两条时的折叠、离散分页、页码与动画。

稳定边界保持不变：occurrence 仍是最小内容身份；查询层不理解词典 DOM；Vue 不反向解析原始词典 HTML。

## 2. 审计方法

### 2.1 原始 HTML

数据库的 `definition_offset/definition_length` 是 UTF-8 **字节偏移**。审计脚本先解压 zlib block，再按字节切片，最后解码 UTF-8：

```powershell
python scripts/audit_dictionary_detail_formats.py `
  --output .agents/analysis/dictionary-detail-audit-20260719/format_audit.json
```

首轮临时扫描曾错误地在已解码字符串上按字符切片，导致词头与正文错位；该批数字已废弃。本文件只使用修正后的脚本输出。

### 2.2 当前与重构后 IR

每个代表词条均使用完整 lookup CLI 固化 JSON、结构化 HTML 与原始 HTML：

```powershell
cargo run -p kotoclip-core --bin kotoclip-cli -- dict-bubble-html `
  --word 振り返る --reading フリカエル `
  --json .agents/analysis/dictionary-detail-audit-20260719/after/furikaeru.lookup.json `
  --output .agents/analysis/dictionary-detail-audit-20260719/after/furikaeru.lookup.html `
  --no-open
```

同批还包括 `鍵`、`あくどい`、`キック`、`立つ`、`仰ぐ`。审计时逐项比较：

```text
原始 HTML
  → 当前适配器字段
  → 兼容 definition_html
  → CLI 可见预览
  → Vue 实际组件结构
```

### 2.3 可见与交互验证

使用本地 HTTP 预览实际检查：

- 解释范围、限定语、标签、中文与分号是否处于同一基线；
- 分号计算颜色是否与正文一致；
- occurrence 按钮是否只使用 occurrence 表头身份，不读取义项正文；
- 例句上一页、下一页、`x/n`、展开和折叠状态是否正确。

## 3. 原始格式族证据

### 3.1 小学馆

审计记录数：94,266。

| 格式族 | 数量 | 代表样本 | 结论 |
| --- | ---: | --- | --- |
| 括号内嵌 `<b>` | 8,690 | `振り返る`、`あの`、`あくどい` | 不能把每个 `<b>` 当作独立译语；括号与逗号必须按源顺序保留。 |
| 同段多个分号 | 5,683 | `AV教育`、`世に出る`、`あくどい` | 分号是组内子句边界，不是新义项，也不能统一改写为逗号。 |
| 行内结构标签 | 6,624 | `成語`、`書面語`、`口語`、`方言` | 标签可能位于中文之后，必须保留前置/后置位置。 |
| 多个日文方括号限定 | 3,558 | `鍵`、`あくどい`、`あの人` | `［日文 A1］中文 A1；［日文 A2］中文 A2` 属于同一解释组的多个子句。 |
| 相同 level/no 连续解释段 | 98 | `キック`、`シート`、`アウト` | 相同义项编号下不同 `〔日文范围〕` 必须合并为多个解释组，不能重复显示相同义项编号。 |
| `〈法〉` 领域标签 | 490 | `エストッペル` 等 | 用户所描述的“法”在该源中实际使用 `〈法〉`，应规范为领域标签。 |
| `成語` 文本出现 | 6,986 | `五月の鯉の吹き流し` 等 | 主要结构为 `.white-square`，应作为行内标签保留。 |

行内标签的结构元素统计为：`成語` 5,576、`書面語` 909、`口語` 552、`方言` 307、`俗語` 78；其余数字标签主要承担局部编号，不能一概视为语域。

### 3.2 三省堂

审计记录数：272,161。

| 格式族 | 词条数 | 代表样本 | 风险 |
| --- | ---: | --- | --- |
| `（動<span class="small">…` 词性行 | 9,500 | `立つ`、`仰ぐ` | 渲染 HTML 后再按纯文本删除会失败，产生字面 `<small>` 或重复词性。 |
| `可能` section | 1,533 | `仰ぐ`、`愛す` | 未截断时可能形会串入义项正文。 |
| `派生` section | 1,507 | `愛くるしい`、`曖昧` | 派生词与词性可能被拼成普通解释。 |
| 多个 invert-rect 主组 | 2,362 | `立つ` | 不同活用组可有各自词性；附加模块必须与最后一个主组分离。 |
| `.no` 带圈数字 marker | 109,314 | `ぱあっと`、`立つ` | 原文使用 ①–㊿；若直接写入统一圆形 marker 容器，会形成双重圈号并与其他层级不一致。 |

`立つ` 的旧 IR 中，第二主组 definition 曾包含：

```text
（動<small>タ</small>下二） 慣用… 表記…
```

重构后第二主组只保留 `動タ下二` 标签与实际义项；`可能形`、`惯用表达`、`表记用法` 分别进入 section。

### 3.3 Crown

审计记录数：33,227。

- 1,803 条同时含 `mean_kubun` 与 `mean_yakugo`，说明“日文范围 + 中文译语”同行是稳定结构，不是个别词条特例；代表样本为 `振り返る`、`鍵`。
- 9,325 条包含超过两条例句，公用例句分页不能只为小学馆设计。

## 4. 代表样本对照

### 4.1 `振り返る`：括号、逗号与连续文本

原始小学馆 HTML：

```html
<span type="語義区分2">〔後ろを〕</span>
<b>回头看</b>，<b>回过头去</b>（<b>看</b>），<b>向后</b>（<b>看</b>）．
```

旧结果：

- gloss 被拆为 `回头看 / 回过头去 / 看 / 向后`；
- definition 残留 `（），（看）`；
- CSS 为拆开的 gloss 重新生成蓝灰色逗号。

新结果：

```text
解释组 〔後ろを〕
  子句 回头看，回过头去（看），向后（看）
```

原始逗号与括号保留在同一连续子句中，前端不再伪造分隔符。

### 4.2 `鍵`、`あくどい`：内部限定与分号

`鍵` 原文：

```html
〔ドアなどの〕［キー］<b>钥匙</b>；［錠前］<b>锁</b>．
```

新 IR：

```text
解释组 〔ドアなどの〕
  子句 [キー] 钥匙
  ；子句 [錠前] 锁
```

`あくどい` 同样形成 `[色が] 过于浓艳；[味が] 太腻`，不会把限定语拆成单独一行。

### 4.3 `キック`：同一义项的多个解释组

原始记录中存在连续的同 `level="2" no="1"` 段：

```html
<p ... no="1">...〔球技などで〕...</p>
<p ... no="1">...〔反動・反発〕...</p>
```

新规则只合并**连续、同 level、同显式 marker** 的段，并将两个 `〔…〕` 保存为同一 sense 下的两个 `gloss_groups`。无 marker 的普通段不会被批量合并。

### 4.4 `立つ`：词性泄漏、附加模块与切换胶囊

修复前：

- `（動<small>タ</small>下二）` 进入正文；
- `慣用/表記/可能` 与第二主组拼接；
- 同形同音且表头元数据不足时，occurrence 胶囊递归读取第一条 gloss/definition 作为身份标签；
- `立つ` 两个真实 occurrence（`128321`、`128322`）的 `display_form/reading` 相同，且 `pos_tags/usage_tags` 均为空，因此稳定进入该分支；
- 生成的两个标签分别以“座ったり横になったり…”和“和船で，各種の柱の称。”开头，与下方义项正文重复。按钮溢出只是该数据串入后的表现。

修复后：

- 在 DOM 节点层识别并剥离主组开头的词性行；
- 在首个 `表記/慣用/派生/可能/補足/注意` deco 前截断正文；
- 附加模块仍由既有 section 适配逻辑消费；
- occurrence 身份标签只允许使用 `display_form/reading`、表头词性/用法标签与 `entry_kind`；
- 同形同音条目缺少可唯一判别的表头元数据时显示 `立つ · 同形条目 1/2`，不再以任何 sense、definition 或 section 内容兜底；
- 按钮宽度与省略样式只作为布局保护，不承担语义纠错。

### 4.5 Crown

`振り返る` 的 `〚後ろを〛 + 回首，回头` 与 `鍵` 的限定语、中文译语均使用同一 `.sense-gloss-group`。Crown 不需要新增源格式规则，只需让兼容 `heading/glosses` 渲染路径采用同行布局。

## 5. 重构方案

### 5.1 统一 IR 扩展

`DictionarySense` 新增：

```text
gloss_groups[]
  ├─ heading?
  └─ clauses[]
       ├─ separator?
       ├─ qualifier?
       ├─ leading_tags[]
       ├─ text
       └─ trailing_tags[]
```

旧 `heading/glosses` 继续生成，供兼容消费者和旧数据使用；Vue 与兼容 HTML renderer 优先读取 `gloss_groups`。

### 5.2 小学馆 token 化

适配器按 DOM 顺序处理：

- `span[type="語義区分2"]` 或顶层含假名的 `〔…〕`：新解释组；
- `［…］`：组内限定；
- `〈…〉`：领域标签；
- `.white-square/.black-square`：行内标签或 marker；
- 顶层分号：新子句，并保存为后一个子句的前置 separator；
- 逗号、括号、嵌套 `<b>`：作为连续文本保留；
- relation anchor：由既有关系提取消费，不混入 gloss 文本。

该规则不依赖词头，不包含代表样本专用分支。

### 5.3 三省堂节点边界

- `strip_leading_pos_line`：只在主组首行整体匹配词性模式时剥离；
- `truncate_at_auxiliary_section`：只识别已知 section deco；
- `.no` 中的 ①–㊿ 在进入 IR 时规范化为 `1–50`，汉字主组与片假名 marker 保持原值；
- 主义项、子义项、例句和 relation 继续使用原解析路径。

### 5.4 公用例句组件

`DictionaryExamples.vue` 同时接入 sense 与 section：

- 两条及以下直接显示；
- 折叠态每页两条；
- `x/n` 位于例句容器上方中央；
- 前后页使用容器左右两侧的全高渐变热区，正文为热区预留等宽边距，不被遮挡或截获；
- 展开态显示全部，支持折叠；
- 页面切换和展开/折叠使用短动画，并尊重 `prefers-reduced-motion`；
- CLI 静态预览生成等价控制器，便于脱离应用验证。

## 7. 容错边界

- 只有连续、同 level、同显式 marker 才合并为同一义项的多个解释组；
- `〔…〕` 只有含日文假名且处于顶层时才可开启新组，中文替换项如 `他〔她〕` 保持普通文本；
- 未识别标签保留为通用 usage 标签，不删除原文；
- 物理宽度不足时允许浏览器自然换行，但不再因组件块级结构产生人为换行；
- 结构化解析失败仍保留既有安全 fallback，不伪造成功。
